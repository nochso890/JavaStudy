# Java ??
- 썬마이크로시스템즈 에서 개발한 객체지향 프로그램 언어 이다.
- 운영체제에 독립적 으로 운영체제 종류에 관계없이 실행가능 하다.


# Java 역사
- Java는 Oak(오크)라는 언어에서부터 시작 되었다.
- Java는 제임스 고슬링 과 아서 밴호프와 같은 썬의 엔지니어 들에 의해 개발 되었다.
- Java의 원래 목표는 가전제품에 탑재 될 소프트웨어를 만드는 것이었다.
- Java의 정식 버전 발표 당시 자바로 작성된 Applet(애플릿)은 정적인 웹 페이지에 사운드 와 애니메이션 등 멀티적인 요소를 제공 할 수 있는 유일한 방법이였다.
- 보안상의 이유로 최신 웹브라우저에서는 더이상 지원하지 않게 되었다.

# Java 특징
- 운영체제에 독립 적이다.
  - JVM(자바가상머신)을 통해 OS에 종속 적이지 않다. 하지만 JVM의 경우는 운영체제에 종속적이다.
  
- 객체지향 언어이다.
  - 객체지향의 특징인 상송, 캡슐화, 다형성이 잘 적용된 객체지향언어라는 평가를 받고 있다.

- 자동 메모리 관리(Garbage Collection)
  - 가비지컬렉터 가 자동적으로 메모리 관리를 한다.

- 멀티쓰레드를 지원한다.
  - 시스템과 관계없이 구현가능 하며, 쓰레드에 대한 스케쥴링을 자바의 인터프리터가 담당 한다.

- 동적 로딩을 지원한다.
  - 실행시 모든 클래스가 로딩되지 않고 필요 시점에 필요 클래스만 로딩된다.
  

# Java 설치
- Java 하기위해서는 JDK 를 먼저 설치해야 한다.
  - JDK 설치하면 JVM 과 Java Class 라이브러리 외에 Java 개발에 필요한 프로그램이 설치된다.
  - JDK 설치가 완료 되면, 설치된 디렉토리에 bin 디렉토리를 환경설정 Path에 추가해야 한다.
  - JDK 의 bin에 있는 주요 실행파일.
    - Javac.exe : 자바 컴파일러, 자바소스코드를 바이트코드로 컴파일 한다.
    - Java.exe : 자바 인터프리터, 컴파일러가 실행한 바이트 코드를 해석한다.
    - Javap.exe : 역어셈블리어, 컴파일된 클래스파일을 원래 코드로 변환 한다.
    - Javadoc.exe : 자동문서 생성기, 소스코드에있는 주석을 통해 API와 같은 문서 같은 형식을 만든다.
    - Jar.exe : 압축프로그램, 클래스파일과 프로그램의 실행에 관련된 파일을 하나의 jar형식으로 압축하거나 해제한다.
  - 설치가 완료되면, cmd 창에서 java --version 입력시 Java version을 확인 가능하다.
